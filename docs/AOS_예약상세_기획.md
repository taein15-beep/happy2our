# AOS 예약상세 기획

## 목적
예약상세 화면은 예약접수현황에서 예약 1건을 선택했을 때 진입하는 상세 관리 화면입니다.

예약 기본정보, 예약자 정보, 여행자 정보, 상품/출발일 정보, 인원/요금, 결제/입금, 옵션, 알림톡 발송내역, 취소/환불, 판매사/공급사 정보, 관리자메모, 첨부파일, 변경이력을 한 화면에서 확인하고 관리합니다.

## 핵심 원칙
- 예약상태와 결제상태는 반드시 분리합니다.
- 판매사 예약과 자사몰 예약을 구분하여 표시합니다.
- 공급상품 예약은 공급사와 판매사의 권한을 구분합니다.
- 예약 발생 후 요금 변경은 기존 예약금액에 자동 반영하지 않습니다.
- 예약상세는 운영자가 빠르게 확인하고 처리할 수 있도록 요약 정보와 상세 정보를 함께 제공합니다.

## 화면 구성

### 1. 예약 요약 상단
목적: 예약의 현재 상태와 금액, 출발 정보를 한눈에 확인합니다.

| 필드명 | 필수/선택 | 화면 노출 방식 | 1차 MVP | 2차 고도화 |
|---|---|---|---|---|
| 예약번호 | 필수 | 상단 요약 텍스트 | 포함 | - |
| 예약상태 | 필수 | 컬러 뱃지 | 포함 | 상태 변경 버튼 연동 |
| 결제상태 | 필수 | 컬러 뱃지 | 포함 | 결제 처리 연동 |
| 예약접수일 | 필수 | 텍스트 | 포함 | - |
| 출발일 | 필수 | 텍스트 | 포함 | 출발 임박 강조 |
| 예약자명 | 필수 | 텍스트 | 포함 | 예약자 상세 링크 |
| 총인원 | 필수 | 숫자 | 포함 | 인원 변경 이력 연동 |
| 총금액 | 필수 | 금액 | 포함 | 금액 산출 상세 |
| 결제금액 | 필수 | 금액 | 포함 | 결제내역 연동 |
| 미수금 | 필수 | 금액, 강조 표시 | 포함 | 미수 알림 발송 |

### 2. 예약 기본정보
목적: 예약 접수 경로와 담당자, 요청사항을 관리합니다.

| 필드명 | 필수/선택 | 화면 노출 방식 | 1차 MVP | 2차 고도화 |
|---|---|---|---|---|
| 예약번호 | 필수 | 읽기 전용 텍스트 | 포함 | - |
| 예약접수일 | 필수 | 읽기 전용 텍스트 | 포함 | - |
| 예약경로 | 필수 | 셀렉트 또는 뱃지 | 포함 | 채널별 권한 연동 |
